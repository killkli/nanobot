# Release notes — v0.1.4.post6

Nanobot v0.1.4.post6 is a release focused on architecture cleanup, provider hardening, end-to-end streaming, and safer day-to-day operations.

---

## Highlights

- **Removed LiteLLM from the runtime path** and switched to native provider integrations plus registry-driven OpenAI-compatible routing
- **Shipped end-to-end streaming improvements** across the agent runner and channel delivery flow
- **Expanded Weixin / WeChat support** and improved cross-channel media handling
- **Improved Feishu streaming behavior** with CardKit-oriented updates
- **Added and documented operational improvements** such as configurable timezones, `/status`, and unified channel login flows
- **Included a security-focused cleanup** related to the `litellm` supply-chain incident

---

## What changed

### Architecture and providers

This release continues Nanobot's move toward a cleaner provider layer:

- direct Anthropic handling through the native Anthropic integration
- shared OpenAI-compatible routing for OpenAI-compatible providers
- better registry-driven provider selection and defaults
- removal of LiteLLM as an active runtime dependency path

Why this matters:

- fewer moving pieces in the request path
- clearer provider behavior and easier debugging
- tighter control over provider-specific parameters and compatibility

### Streaming and agent runtime

Streaming behavior received several updates in this release window:

- end-to-end streaming support was improved
- stream delta coalescing was refined at boundaries
- agent runner lifecycle hooks were unified
- agent runner internals were extracted and decoupled for clearer architecture

Why this matters:

- more stable progressive output
- fewer duplicate or awkward partial updates in channels
- cleaner internal execution flow for future feature work

### Channels and login flows

Channel support also moved forward:

- Weixin / WeChat support and compatibility improved
- WhatsApp and WeChat media handling were refined
- Feishu streaming behavior improved with CardKit-oriented updates
- interactive channel login flows became more unified and better documented

Why this matters:

- easier onboarding for channel-based deployments
- more reliable media and streaming behavior in real chat environments
- less guesswork when authenticating channels from the CLI

### Configuration and operations

This release also includes several user-facing quality-of-life improvements:

- configurable agent timezone support
- `/status` command support in the release window
- clearer docs for OAuth providers and local/self-hosted model setups

Why this matters:

- easier deployment in teams and multiple regions
- better operational visibility
- more predictable behavior for scheduled tasks and runtime context

---

## Security note

Due to the `litellm` supply-chain poisoning incident, Nanobot removed `litellm` from the active runtime path in this release line.

If you previously used environments that installed older Nanobot versions or pulled LiteLLM-related dependencies separately, review your Python environment and follow the project's advisory guidance before continuing production use.

## Upgrade notes

### Upgrade from PyPI / pip

```bash
pip install -U nanobot-ai
nanobot --version
```

### Upgrade from uv

```bash
uv tool upgrade nanobot-ai
nanobot --version
```

Expected version:

```bash
nanobot 0.1.4.post6
```

### WhatsApp users

If you use WhatsApp, rebuild the local bridge after upgrading:

```bash
rm -rf ~/.nanobot/bridge
nanobot channels login whatsapp
```

---

## Recommended follow-up checks

After upgrading:

1. confirm `nanobot --version` reports `0.1.4.post6`
2. review your configured providers and remove stale LiteLLM assumptions from local workflows
3. verify channel logins still work as expected, especially for Weixin and WhatsApp
4. test streaming behavior in your primary channel
5. review timezone-related settings if you rely on cron or schedule execution
